# Governing Intelligent Systems

Software is becoming more autonomous, more connected, and more capable of acting through real-world systems.

That changes the work.

The hard question is no longer only whether a system can be built. It is whether it can be governed as it changes, connects, adapts, and drifts.

## Core Idea

Intelligent systems need more than execution environments. They need operational coherence:

- clear control surfaces
- explicit authority boundaries
- provenance
- reviewable change
- recoverability
- accountability
- practical governance that supports progress

The reason is simple: [Because Drift Happens™](https://becausedrifthappens.com/).

## Working Vocabulary

- **Drift**: the natural movement of a system away from its intended state, assumptions, boundaries, or operating model.
- **Operational coherence**: the maintained alignment between intent, behaviour, controls, and consequences.
- **Control surface**: the set of places where a system can act, be acted upon, escalate, mutate, or affect the outside world.
- **Provenance**: evidence of where content, decisions, artefacts, and system states came from.
- **Governance**: the operating discipline that keeps systems coherent under change.

# Machine-Readable Publishing

This vault is based on a bet: public content should be easy for people, search engines, and AI systems to inspect without needing to infer structure from a CMS.

The durable source is Markdown in Git. The website is the public rendering of that source.

## Working Principles

- Keep the canonical source in plain text.
- Use stable, descriptive paths.
- Add metadata when it helps discovery.
- Prefer simple linking over platform-specific publishing features.
- Make generated outputs, feeds, and discovery files easy to find.
